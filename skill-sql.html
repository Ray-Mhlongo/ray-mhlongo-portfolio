Querying packet data like a database

I used the provided Wireshark CSV export as a SQL analytics project. The goal was to clean packet rows, group protocol activity, rank busy hosts, and explain the result in a way a technical team can scan quickly.

Back Home

Project Structure

Business Problem

A support team needs to know which protocols and hosts dominate a network capture before troubleshooting deeper.

Dataset Overview

The main dataset contains 394,136 packet records with time, source, destination, protocol, packet length, and packet information.

Solution Approach

I modeled the CSV as relational tables: packets, hosts, protocol summary, and alerts.

Technologies Used

SQL logic, CTEs, CASE statements, GROUP BY, JOIN thinking, window functions, and CSV data cleaning.

CTE, CASE, and window ranking

The top source was 192.167.7.162 with 116,067 packets, making it the first host I would investigate for normal behavior or heavy usage.

Final Outcome

Key Insight

TCP and TLSv1.3 dominate the capture, so the first review should focus on session volume and encrypted traffic patterns.

Data Cleaning

Blank protocol and host values are filtered before aggregation so summary results do not mislead the reader.

Business Value

The SQL layer turns packet rows into clear investigation priorities.
