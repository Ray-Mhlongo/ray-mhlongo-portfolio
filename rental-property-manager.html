Project Case Study

Rental Property Manager - Real Estate Operations Analytics

A structured analytics system for monitoring rental portfolio performance, tenant activity, and revenue collection.

This project demonstrates how operational data from a rental property portfolio can be structured and analyzed to support better property management decisions.

Using Excel-based data modeling, I built a system that tracks properties, tenants, rental payments, and maintenance expenses while generating operational insights about occupancy, arrears, and portfolio performance.

The objective was to transform scattered property records into structured operational analytics that provide landlords with clear visibility into revenue, vacancies, and property costs.

Confidentiality note: this case study uses demo or anonymized data for portfolio purposes.

Problem

Managing multiple rental properties manually makes it difficult to monitor key operational metrics such as occupancy rates, rent collection performance, lease expirations, and maintenance costs.

When data is spread across separate spreadsheets or informal records, landlords cannot easily answer critical business questions such as:

  • Which properties are generating the most revenue?
  • Which tenants have outstanding rent balances?
  • When leases are expiring
  • How much is being spent on property maintenance

Without structured reporting, it becomes difficult to monitor the financial health of a property portfolio.

Solution

I developed a centralized rental property analytics workbook that organizes property, tenant, payment, and maintenance data into structured tables.

The system automatically calculates important operational indicators such as occupancy rates, rent arrears, and maintenance costs per property. These metrics allow landlords to quickly monitor portfolio performance and identify operational risks.

Data Organized and Tracked

  • Property Portfolio: property ID and location, number of units or rooms, rental pricing information
  • Tenant Data: tenant identification, lease start and end dates, deposit records
  • Rental Payments: monthly rent payments, payment status (Paid / Outstanding), payment method
  • Maintenance Records: maintenance job descriptions, service providers, cost tracking, completion status
  • Operating Expenses: expense categories per property, monthly property operating costs

Data Analyst Responsibilities

  • Designing a structured data model linking properties, tenants, payments, and maintenance records
  • Standardizing data fields such as dates, payment status categories, and cost entries
  • Creating operational KPI calculations for occupancy rate and rent arrears
  • Using Excel formulas to automate reporting and financial tracking
  • Developing summary reporting views for property portfolio performance

Key Metrics Tracked

The demonstration dataset includes:

  • 7 properties within the portfolio
  • 50+ tenant records
  • 50+ rental payment transactions

Key operational metrics tracked include:

  • Occupancy rate per property
  • Outstanding rent balances
  • Lease expiration pipeline
  • Maintenance spending per property
  • Rent revenue performance

Tools Used

  • Microsoft Excel
  • Data modeling and structured tables
  • Excel analytical formulas
  • Operational KPI reporting
  • Property portfolio analytics

Business Impact

By consolidating operational property data into a structured analytics system, landlords gain clearer visibility into their rental portfolio performance.

The system helps prioritize actions such as addressing vacancies, following up on unpaid rent, and monitoring property maintenance costs, ultimately improving operational oversight and financial control.
