This project helps people compare internet plans in a simple and practical way. It shows which plan gives the best value for money, based on speed and monthly cost.

Project Overview

This project compares internet plans from providers that serve one small South African town. It is designed for residents, small businesses, and anyone who wants to avoid overpaying for internet. The main purpose is to make plan choices clear, fair, and easy to understand.

The Problem (Why this project exists)

Many people pick an internet provider out of habit, ads, or brand name. In real life, that can mean paying more for less speed. For example, two homes in the same town can pay different prices for similar internet quality. Without a clear comparison, it is hard to know which plan is truly worth it.

The Solution (How my system helps)

I built a simple comparison process focused on what users care about most.

  • I collected plan details from public provider websites for this town only.
  • I listed key details like speed, monthly price, and setup cost in one place.
  • I used one simple measure: Cost per Mbps (price divided by speed).
  • I marked the best-value plan for each provider.
  • I shared the results in a dashboard so users can compare plans quickly.

In simple terms, users can open the dashboard, compare plans side by side, and choose a plan with better value.

Key Features

  • Clear plan comparison for the same town.
  • Best-value highlight for each provider.
  • Simple view of speed versus monthly cost.
  • Interactive dashboard for fast decision-making.
  • Practical insights for homes and small businesses.

Tools & Technologies Used

  • Microsoft Excel - Used to organize, clean, and calculate the plan data.
  • Power BI - Used to build the interactive dashboard and visual comparisons.
  • Public Provider Websites - Source of package prices, speeds, and setup details.

What I Learned

This project taught me how to turn raw public data into useful, real-world guidance. I improved my skills in data collection, data cleaning, and visual storytelling. I also learned how to present technical findings in plain language for non-technical audiences.
