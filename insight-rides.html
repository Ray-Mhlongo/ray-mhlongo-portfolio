Case Study

Problem

Small transport operators often track students, trips, payments, vehicles, and costs in separate spreadsheets. That makes it hard to see income, outstanding fees, daily trips, and route performance in one place.

What I did

  • Created a dashboard for income, expenses, trips, students, fuel, and alerts.
  • Built pages for students, trips, routes, maps, vehicles, payments, finance, and reports.
  • Added Google Drive sync so records can be stored as structured data files.
  • Added offline saving so operators can keep working when connection is weak.

Tools used

React, Vite, Tailwind CSS, Chart.js, Leaflet maps, Google Drive API, local storage, and structured JSON data.

Key outcome

The project turns daily transport activity into useful business insight. An operator can see income, unpaid students, trip activity, route data, fuel estimates, and reports without digging through separate files.

Data Thinking

Cleaner records

Students, trips, routes, vehicles, payments, income, and expenses are stored as structured records that can be analyzed.

Better visibility

The dashboard highlights daily income, monthly profit, outstanding payments, fuel costs, and route activity.

Actionable alerts

The system flags missing trips, unpaid students, and high fuel estimates so the operator knows what needs attention.
