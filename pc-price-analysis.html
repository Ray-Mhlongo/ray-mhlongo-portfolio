Completed Project

PC Price Analysis

Advanced marketplace indexing using SQL and Power BI to standardize raw PC hardware data for meaningful price comparison insights.

Dashboard

Project Overview

This portfolio project analyzes PC component prices across shops and provinces in South Africa to build practical SQL data modeling and analytics skills. The dataset is cleaned and standardized, then structured for visualization in tools like Power BI. The work is learning-focused but follows analyst-grade discipline.

Tools Used

  • SQL (MySQL)
  • Power BI
  • GitHub

What I Did

  • Designed a relational database to capture PC parts pricing and attributes
  • Inserted raw product data from multiple sources
  • Cleaned and standardized inconsistent names and formatting
  • Built dimension tables for categories, provinces, shops, and manufacturers
  • Applied primary keys and foreign keys to enforce data integrity
  • Ran analytical queries to answer business-style questions

Database Structure

Core table: Parts

Dimension tables: part_categories, Province, Shop, Manufacturer

This structure enables filtering by category (CPU, GPU, RAM, etc.), province, shop, and manufacturer for consistent comparison.

Key Questions Answered

  • What products are available in each province?
  • What are the cheapest products per category?
  • How do prices differ between shops?
  • Which products have the highest warranties?
  • What are the most and least expensive items per category?

Folder Structure

  • /scripts/ → SQL scripts to create tables and insert data
  • /data/ → CSV files of raw or cleaned data
  • /PowerBI/ → Power BI files and dashboards
