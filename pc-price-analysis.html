Project Case Study

PC Parts Price Analysis (South Africa)

This learning-focused portfolio project analyzes PC component prices across different shops and provinces in South Africa. The goal was to model clean, analysis-ready data that can support SQL insights and Power BI reporting.

Tools Used

  • SQL (MySQL)
  • Power BI
  • GitHub

What I Did

  • Created a relational database for PC parts pricing.
  • Inserted raw product-level data from multiple sources.
  • Cleaned and standardized inconsistent names and formatting.
  • Built dimension tables for categories, provinces, shops, and manufacturers.
  • Applied primary and foreign key relationships for data integrity.
  • Ran analytical SQL queries to answer business-style pricing questions.

Database Structure

Main table: Parts

Dimension tables:

  • part_categories
  • Province
  • Shop
  • Manufacturer

This structure supports filtering by category (CPU, GPU, RAM, etc.), province, shop, and manufacturer.

Key Questions Answered

  • What products are available in each province?
  • What are the cheapest products per category?
  • How do prices differ between shops?
  • Which products have the highest warranties?
  • What are the most and least expensive items per category?

Folder Structure / Resources

  • /scripts/ → SQL scripts for schema creation and data inserts
  • /data/ → Raw and cleaned CSV data files
  • /PowerBI/ → Power BI report files and dashboard outputs

Optional SQL / Analysis Snippet

-- Example analysis query snippet
SELECT c.category_name,
       MIN(p.price) AS cheapest_price,
       MAX(p.price) AS highest_price
FROM Parts p
JOIN part_categories c ON p.category_id = c.category_id
GROUP BY c.category_name
ORDER BY cheapest_price ASC;
