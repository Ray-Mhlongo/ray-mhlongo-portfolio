Case Study

Short version: raw product prices were messy, so I cleaned and structured them for fair comparison.

Problem

PC component prices were difficult to compare because product data came from different shops and locations.

What I Did

  • Cleaned raw hardware pricing data.
  • Standardized product categories, shops, provinces, and manufacturers.
  • Created a relational SQL structure for analysis.
  • Built dashboard views for product and price comparison.

Tools Used

SQL MySQL Power BI GitHub

Key Outcome

The project made it easier to identify low-cost products, compare shops, and spot price differences by category and province.

Key Insights

The analysis turned messy product listings into clearer price comparison answers.
